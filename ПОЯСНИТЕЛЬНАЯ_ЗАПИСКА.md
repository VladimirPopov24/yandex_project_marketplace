# Пояснительная записка
## Проект «Modex Marketplace» — учебный маркетплейс на Flask

---

## 1. Общие сведения

**Название проекта:** Modex Marketplace  
**Тип:** Веб-приложение (полный цикл — бэкенд + шаблонный фронтенд)  
**Назначение:** Учебный проект в рамках курса Яндекс Лицея — реализация многоролевого маркетплейса товаров.

---

## 2. Цель и задачи

**Цель:** Разработать функциональный интернет-магазин с системой ролей, каталогом товаров, корзиной, избранным, оформлением заказов и административной панелью.

**Задачи:**
- реализовать аутентификацию и авторизацию на основе JWT;
- обеспечить разграничение прав между покупателем, продавцом и администратором;
- реализовать каталог товаров с фильтрацией и поиском;
- добавить функционал корзины и оформления заказа;
- создать панель продавца для управления собственными товарами;
- создать административную панель с модерацией товаров и управлением пользователями.

---

## 3. Стек технологий

| Слой | Технология | Обоснование |
|------|-----------|-------------|
| Веб-фреймворк | Flask | Высокая производительность, автоматическая генерация OpenAPI-документации, встроенная поддержка async |
| ASGI-сервер | Uvicorn | Стандартный сервер для FastAPI, поддержка WebSocket и HTTP/2 |
| ORM | SQLAlchemy | Гибкая работа с реляционными БД, декларативные модели |
| База данных | SQLite | Простое развёртывание без отдельного сервера, достаточно для учебного проекта |
| Аутентификация | python-jose (JWT) | Токены хранятся в cookies, не требуют серверного хранилища сессий |
| Хеширование паролей | Werkzeug | Надёжные алгоритмы (PBKDF2/scrypt), простой API |
| Шаблонизатор | Jinja2 | Нативная интеграция с FastAPI, мощные возможности наследования шаблонов |
| Загрузка файлов | python-multipart | Обработка multipart/form-data для загрузки изображений товаров |

---

## 4. Архитектура приложения

Приложение построено по принципу **маршрутизированных модулей** (Router-based architecture):

```
app/
├── main.py          — точка входа, регистрация роутеров, seed начальных данных
├── database.py      — подключение к SQLite, фабрика сессий
├── models.py        — ORM-модели (таблицы БД)
├── auth.py          — JWT-токены, хеширование, получение текущего пользователя
├── routers/
│   ├── auth.py      — регистрация, вход, выход
│   ├── products.py  — каталог и страница товара
│   ├── cart.py      — корзина и оформление заказа
│   ├── favorites.py — избранное
│   ├── seller.py    — дашборд продавца
│   └── admin.py     — панель администратора
├── templates/       — HTML-шаблоны Jinja2
└── static/
    └── uploads/     — загруженные изображения товаров
```

**Поток запроса:**  
Браузер → Uvicorn → Flask Router → Зависимости (get_db, get_current_user) → Бизнес-логика → SQLAlchemy → SQLite → Jinja2-шаблон → HTML-ответ

## 5. Модель данных

### Диаграмма связей

```
User ──┬── CartItem ── Product ── Category
       ├── Favorite  ──┘
       └── Order
```

### Описание таблиц

| Таблица | Поля | Назначение |
|---------|------|-----------|
| `users` | id, username, password_hash, role, is_blocked, created_at | Пользователи системы |
| `categories` | id, name | Категории товаров |
| `products` | id, name, description, price, category_id, seller_id, img_url, status, created_at | Товары (статус: pending / approved / rejected) |
| `cart_items` | id, user_id, product_id, quantity | Позиции в корзине |
| `favorites` | id, user_id, product_id | Избранные товары |
| `orders` | id, user_id, status, total, created_at | Оформленные заказы |

---

## 6. Система ролей

| Роль | Права |
|------|-------|
| `buyer` | Просмотр каталога, корзина, избранное, оформление заказа, переход в продавца |
| `seller` | Всё права покупателя + управление собственными товарами (CRUD, загрузка фото) |
| `admin` | Полный доступ: статистика, модерация товаров, управление пользователями и категориями |

## 7. Функциональные возможности

### 7.1 Аутентификация
- Регистрация и вход по логину / паролю
- Пароли хранятся в виде хеша (Werkzeug PBKDF2)
- Сессия хранится в cookie-токене JWT (время жизни — 24 часа)
- Заблокированные пользователи не могут войти в систему

### 7.2 Каталог товаров
- Постраничный вывод одобренных товаров
- Поиск по названию и описанию
- Фильтрация по категории
- Фильтрация по диапазону цен
- Сортировка (по цене, новизне)
- Детальная страница товара с блоком похожих товаров

### 7.3 Корзина
- Добавление товара через AJAX (без перезагрузки страницы)
- Изменение количества позиций
- Удаление позиций
- Отображение итоговой суммы в реальном времени

### 7.4 Оформление заказа
- Форма с именем, телефоном, адресом доставки и способом оплаты
- При оформлении создаётся запись `Order`, корзина очищается
- Страница подтверждения с номером заказа

### 7.5 Избранное
- Добавление / удаление через AJAX
- Счётчик избранного в шапке сайта

### 7.6 Кабинет продавца
- Просмотр своих товаров с указанием статуса модерации
- Добавление товара (название, описание, цена, категория, фото)
- Редактирование и удаление товаров
- Загрузка изображения с сохранением на диск (форматы: jpg, jpeg, png, webp, gif)

### 7.7 Административная панель
- Дашборд: статистика (пользователи, товары, заказы, выручка)
- Очередь модерации: одобрить / отклонить / удалить товар продавца
- Управление пользователями: поиск, блокировка / разблокировка, смена роли
- Управление категориями: добавление и удаление

---

## 8. Безопасность

- Пароли никогда не хранятся в открытом виде — только PBKDF2-хеш
- JWT подписан секретным ключом (алгоритм HS256)
- Доступ к страницам продавца и администратора защищён проверкой роли на сервере
- Загрузка файлов: проверяется расширение и генерируется случайное UUID-имя, что исключает path traversal и перезапись чужих файлов

## 9. Установка и запуск

```bash
# 1. Клонировать репозиторий
git clone <url>
cd yandex_project_marketplace

# 2. Создать виртуальное окружение
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Запустить сервер
uvicorn app.main:app --reload
```

Приложение будет доступно по адресу: **http://localhost:8000**

> При первом запуске база данных создаётся автоматически и заполняется тестовыми данными: 14 категорий и 40+ товаров с реальными изображениями.
